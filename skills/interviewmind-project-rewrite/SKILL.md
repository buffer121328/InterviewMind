# STAR 法项目改写

## 启动

1. 如果被 `interviewmind-resume` 调用 → 直接读取 `mianbiguo/resume.md` 中项目部分
2. 如果是独立使用 → 请用户提供一段项目经历原文（从简历中复制即可）
3. 读取 `skills/interviewmind/references/star-template.md` 获取改写模板

## 改写流程

### 步骤 1: STAR 拆解

对每个项目输出拆解表：

| STAR 要素 | 原文中的内容 | 缺失度 |
|-----------|-------------|--------|
| 情境 | [从原文提取] | ✓/✗+缺少什么 |
| 任务 | [从原文提取] | ✓/✗+缺少什么 |
| 行动 | [从原文提取] | ✓/✗+缺少什么 |
| 结果 | [从原文提取] | ✓/✗+缺少什么 |

### 步骤 2: 缺口填补

对标记 ✗ 的要素，逐个问用户。例如：

- "原文没有提到这个项目的具体结果。上线后QPS/延迟/用户量有什么变化？即使大概的数字也好。"
- "你在这个项目里的具体任务是什么？是独立完成核心模块还是负责协调团队？"

不要编造数据 — 用户说不知道的部分，用合理描述替代（如"参与设计高可用架构"而非虚构"QPS 提升 200%"）。

### 步骤 3: 重写

每条 bullet point 满足：
- 强动词开头（设计、实现、优化、重构、主导、搭建、迁移、降低、缩短）
- 包含量化数据（数字、百分比、规模、时长）
- 单条 ≤ 80 中文字符
- 禁止 "参与了" "协助了" "负责了" "做了"

### 步骤 4: 输出

输出到 `mianbiguo/outputs/star-projects-YYYYMMDD.md`：

```markdown
# STAR 项目改写

## 项目: [项目名]

**情境**: [1-2句]
**任务**: [1句]

**关键成果**:
- [star bullet 1]
- [star bullet 2]
- [star bullet 3]
```

如果用户有多个项目需要改写，逐个处理。

## 独立模式 vs 批量模式

- **独立模式**（用户主动调用）：处理完一个项目后问"还有其他项目需要改写吗？"
- **批量模式**（被 resume skill 调用）：跳过确认环节，处理 resume.md 中所有可识别的项目，输出合并到 star-projects-YYYYMMDD.md
