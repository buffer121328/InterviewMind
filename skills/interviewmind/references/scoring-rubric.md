# 简历6维度评分标准

对简历进行逐维度 1-10 分评分，每维度附具体依据。

## 维度定义

### 1. 结构 (Structure)
评估排版、信息层级、模块划分是否清晰。
- 8-10: 模块分明、层级清晰、视觉扫描20秒能定位关键信息
- 5-7: 有基本分区但偶有信息混杂，搜寻某类信息需>30秒
- 1-4: 杂乱无章，无标题区分，段落堆砌

### 2. 完整度 (Completeness)
评估关键信息是否齐全：联系方式、教育背景、工作经历、项目经历、技能清单。
- 8-10: 六要素齐全，每段经历有时间/公司/职位/职责/成果
- 5-7: 缺少1-2个要素，或部分经历信息不完整
- 1-4: 缺少3个以上关键要素

### 3. 量化 (Quantification)
评估数据支撑是否充分（数字、百分比、规模、时长）。
- 8-10: 70%+的 bullet point 包含量化数据
- 5-7: 30-70% bullet point 有数据
- 1-4: <30% bullet point 有数据

### 4. 清晰度 (Clarity)
评估表达是否简洁准确，无模糊词、无歧义。
- 8-10: 每句话有明确主语+动作+结果，无"参与了""协助了"等弱动词
- 5-7: 存在少量模糊表述但整体可理解
- 1-4: 大量"负责相关工作""参与项目开发"等空洞表述

### 5. 亮点 (Highlight)
评估差异化竞争力是否突出：技术深度、业务影响、奖项成就、开源贡献等。
- 8-10: 3个以上突出亮点，有具体成就和数据背书
- 5-7: 1-2个亮点但无深层展开
- 1-4: 无亮点，简历与其他候选人无差异

### 6. JD匹配 (JD Match)
评估与目标岗位要求对齐度：技能栈、行业经验、级别匹配。
- 8-10: 核心技能100%覆盖JD要求，有直接相关项目经验
- 5-7: 部分匹配，有可迁移技能但缺1-2项关键要求
- 1-4: 技能栈与JD要求重叠<40%

## 输出格式

评分时使用以下表格：

| 维度 | 评分(1-10) | 具体依据 |
|------|-----------|----------|
| 结构 | X | 一句话说明扣分/加分原因 |
| 完整度 | X | 缺失哪些关键信息 |
| 量化 | X | 量化数据覆盖率 |
| 清晰度 | X | 模糊表述举例 |
| 亮点 | X | 是否有差异化亮点 |
| JD匹配 | X | 技能重叠度 |

综合建议（3条以内优先修改项）。
