# STAR 法项目改写模板

## STAR 拆解框架

| 环节 | 问自己的问题 | 在原文中怎么找 |
|------|-------------|---------------|
| **S**ituation 情境 | 项目的背景是什么？为什么需要做？ | 业务背景、技术债、性能瓶颈、排期压力 |
| **T**ask 任务 | 你需要完成什么目标？你的职责是什么？ | 需求文档、技术方案、任务分配 |
| **A**ction 行动 | 你具体做了什么？技术选型？关键决策？ | 架构设计、编码实现、问题排查、团队协调 |
| **R**esult 结果 | 项目带来了什么影响？有量化数据吗？ | 上线指标、性能数据、用户增长、团队效率 |

## 改写规则

每条 bullet point 必须：
1. 以强动词开头（设计、实现、优化、主导、重构、迁移、搭建）
2. 包含至少一个量化结果（数字、百分比、规模）
3. 单条不超过 2 行（<80 中文字符）
4. 禁止使用：参与了、协助了、负责了、做了

## 改写示例

原文：
> 参与了电商平台后端开发，主要负责订单模块，性能优化，QPS提升

改写后：
- 设计并实现订单系统分布式事务方案，支撑日均 50 万订单量，数据一致性达 99.99%
- 优化订单查询链路，通过 Redis 多级缓存将 P99 延迟从 800ms 降至 120ms
- 重构订单状态机，消除 4 类并发竞态，线上故障从月均 2 次降为 0

## 输出格式

每个项目输出：

**项目：[项目名称]**
- S: [1-2句情境]
- T: [1句任务]
- A: [3-5条改写后bullet，每条以强动词开头，含量化数据]
- R: [1-2句结果/影响]
