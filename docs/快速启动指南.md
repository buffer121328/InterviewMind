# 🚀 快速启动指南

本指南帮助你快速在本地运行 AI 面试智能体项目。

---

## 前置要求

- **Python 3.11+**
- **Node.js 20+**
- **PostgreSQL 16** (本地安装或 Docker)
- **OpenAI API Key** (或兼容 API)

---

## 第一步：数据库准备

### 方式一：使用 Docker (推荐)

```bash
# 启动 PostgreSQL 容器
docker run -d \
  --name agent_interview_db \
  -e POSTGRES_USER=agent_interview \
  -e POSTGRES_PASSWORD=cheng123 \
  -e POSTGRES_DB=agent_interview \
  -p 5432:5432 \
  pgvector/pgvector:pg16
```

### 方式二：本地 PostgreSQL

确保 PostgreSQL 已安装并运行，创建数据库：

```sql
CREATE DATABASE agent_interview;
CREATE USER agent_interview WITH PASSWORD 'cheng123';
GRANT ALL PRIVILEGES ON DATABASE agent_interview TO agent_interview;
```

---

## 第二步：配置环境变量

在项目根目录创建 `.env` 文件：

```bash
# .env 文件内容

# === 数据库 ===
DATABASE_URL=postgresql://agent_interview:cheng123@localhost:5432/agent_interview

# === LLM 配置 ===
OPENAI_API_KEY=请填你自己的API密钥
OPENAI_BASE_URL=https://your-openai-compatible-base-url/v1
SMART_MODEL=XiaomiMiMo/MiMo-V2-Flash
FAST_MODEL=opencode-go
```

---

## 第三步：启动后端服务

```bash
# 1. 进入后端目录
cd backend

# 2. 使用 uv 创建虚拟环境并安装依赖
uv sync

# 3. 激活虚拟环境
source .venv/bin/activate

# 4. 初始化数据库
python -m app.database.init_db

# 5. 启动服务
python main.py
```

✅ 看到以下输出表示启动成功：

```
AI 面试助手后端服务启动中...
数据库初始化完成
启动服务器: http://0.0.0.0:8000
API 文档: http://localhost:8000/docs
```

---

## 第四步：启动前端服务

```bash
# 新开一个终端窗口

# 1. 进入前端目录
cd web

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev
```

✅ 前端将在 `http://localhost:3000` 启动

## 常见问题

### Q1: 数据库连接失败

**症状**: `Connection refused` 或 `could not connect to server`

**解决方案**:
1. 确认 PostgreSQL 正在运行
2. 检查 `DATABASE_URL` 配置是否正确
3. 确认数据库用户权限

```bash
# 测试数据库连接
psql -h localhost -U agent_interview -d agent_interview
```

### Q2: 后端依赖安装失败

**症状**: `uv sync` 报错

**解决方案**:
```bash
# 确认 uv 已安装
uv --version

# 如果未安装 uv
pip install uv

# 重新同步依赖
cd backend
uv sync

# 如果仍有问题，清除缓存后重试
uv cache clean
uv sync
```

### Q3: 前端无法连接后端

**症状**: API 请求失败

**解决方案**:
1. 确认后端在 8000 端口运行
2. 检查 `web/.env.local` 配置：
   ```
   NEXT_PUBLIC_API_URL=http://localhost:8000
   ```

### Q4: LLM API 调用失败

**症状**: `OPENAI_API_KEY` 相关错误

**解决方案**:
1. 确认 API Key 有效
2. 如使用代理，检查 `OPENAI_BASE_URL` 配置
3. 在前端设置对话框中配置 API

## 开发调试

### 查看后端日志

```bash
# 实时查看输出
python main.py 2>&1 | tee backend.log
```

### 查看数据库内容

```bash
# 连接数据库
docker exec -it agent_interview_db psql -U agent_interview

# 或本地连接
psql -h localhost -U agent_interview -d agent_interview

# 常用 SQL
\dt                           # 查看所有表
SELECT * FROM sessions;       # 查看会话
SELECT * FROM messages LIMIT 10; # 查看消息
```

### 重置数据库

```bash
# 删除所有数据并重新初始化
cd backend
python -c "
from app.database.init_db import reset_database
import asyncio
asyncio.run(reset_database())
"
```

## Docker Compose 快速启动

如果你想一键启动所有服务：

```bash
# 复制环境变量模板
cp .env.production.example .env.production

# 编辑配置
vim .env.production

# 启动所有服务
docker-compose --env-file .env.production up -d --build

# 初始化数据库
docker-compose exec backend python -m app.database.init_db

# 查看日志
docker-compose logs -f
```

访问:
- 前端: `http://localhost` (通过 Nginx)
- API 文档: `http://localhost/api/docs`
