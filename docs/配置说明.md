# 面必过：全量配置说明

> 仓库根目录 [`env_example`](../env_example) 是唯一全量 `.env` 模板。初始化时执行 `cp env_example .env`，然后逐项确认所有变量；不要删除暂时不用的变量，也不要从旧文档拼接配置。功能是否运行由 `*_ENABLED` 等开关控制，变量本身始终保留。

## 1. 全量 `.env` 初始化流程

```bash
cp env_example .env
uv run --project backend python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
openssl rand -hex 32
```

初始化后必须确认：

```dotenv
POSTGRES_PASSWORD=<强密码>
DATABASE_URL=postgresql://agent_interview:<同一密码>@localhost:5432/agent_interview
TASK_PAYLOAD_ENCRYPTION_KEY=<Fernet.generate_key() 的输出>
BOSS_AUTOMATION_SERVICE_TOKEN=<openssl rand -hex 32 的输出，至少 32 字符>
OPENAI_API_KEY=<Embedding/RAG 使用的 OpenAI-compatible Key>
MEM0_LLM_API_KEY=<mem0 记忆提取模型 Key，MEM0_ENABLED=true 时必须可用>
MEM0_EMBEDDER_API_KEY=<mem0 语义检索模型 Key，MEM0_ENABLED=true 时必须可用>
LANGFUSE_PUBLIC_KEY=<LANGFUSE_ENABLED=true 或启用 prompt/eval 上报时必须可用>
LANGFUSE_SECRET_KEY=<LANGFUSE_ENABLED=true 或启用 prompt/eval 上报时必须可用>

# 本机 Web 开发使用此值；Docker Compose 全容器模式使用 /api。
NEXT_PUBLIC_API_URL=http://localhost:8000
```

`TASK_PAYLOAD_ENCRYPTION_KEY` 用于加密持久化 AgentRun payload。API 与 Worker 必须完全一致；只要接口会创建 AgentRun 就需要它，不是仅在队列模式下才需要。不得提交到 Git、日志、截图或示例请求。

对话模型的 API Key、Base URL、模型名称和模型池成员在前端「设置 → 模型配置」维护，浏览器会在业务请求的 `api_config` 中发送。标准验收请完整配置并验证 Smart、Fast、Voice、Reasoning Pool、Fast Pool 与简历专家通道；这些模型密钥不写入服务器 `.env`。

## 2. 配置归属与运行模式

| 配置类别 | 位置 | 是否敏感 | 说明 |
| --- | --- | --- | --- |
| PostgreSQL、Redis、队列、RAG、Embedding、语音、mem0、Langfuse、BOSS | 根 `.env` | 部分敏感 | 复制 `env_example` 后保留全量变量并逐项确认。 |
| Smart/Fast、模型池、Voice、简历专家通道 | 前端「设置 → 模型配置」 | 是 | 保存在浏览器本地，随请求传给后端。 |
| Compose 容器内连接地址 | `docker-compose.yml` | 否 | Compose 会把 API/Worker 的 DB 与 Redis 地址覆盖为服务名。 |
| 部署秘密 | CI/部署平台 Secret | 是 | 生产环境不要通过 Git、镜像层或公开日志保存。 |

| 模式 | `NEXT_PUBLIC_API_URL` | DB/Redis 地址 | 队列 |
| --- | --- | --- | --- |
| 本机代码 + Docker 依赖 | `http://localhost:8000` | `.env` 的 `localhost` | 标准配置为 `TASK_QUEUE_ENABLED=true`，启动 Worker |
| Docker Compose 全容器 | `/api` | Compose 自动改为 `postgres` / `redis` | 标准配置为 `TASK_QUEUE_ENABLED=true`，启动 Worker |

## 3. 变量分组

### 3.1 PostgreSQL 与应用监听

```dotenv
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=agent_interview
POSTGRES_PASSWORD=change_me
POSTGRES_DB=agent_interview
DATABASE_URL=postgresql://agent_interview:change_me@localhost:5432/agent_interview

HOST=0.0.0.0
PORT=8000
DEBUG=false
NEXT_PUBLIC_API_URL=/api
AUTO_CREATE_TABLES=true
```

- `DATABASE_URL` 优先；同时保留 `POSTGRES_*`，供 Compose、mem0 回退和运维排查使用。
- 本机前端直连后端时使用 `NEXT_PUBLIC_API_URL=http://localhost:8000`；Compose 全容器保持 `/api`，由 Nginx 同源代理。
- 生产/Compose 模式使用 Alembic 管理 schema；全新库或已管理库执行 `uv run alembic upgrade head`，部署入口使用 `python -m app.entrypoints.deployment migrate`。
- `DEBUG=true` 仅影响直接运行 `python -m app.main` 时的 Uvicorn reload，不能代替生产调试策略。

### 3.2 Redis、AgentRun 与 Worker

```dotenv
REDIS_PORT=6379
REDIS_URL=redis://localhost:6379/0
TASK_QUEUE_ENABLED=true
TASK_PAYLOAD_ENCRYPTION_KEY=
AGENT_RUN_LOCK_TTL_SECONDS=600
AGENT_RUN_MAX_ATTEMPTS=3
AGENT_RUN_STALE_SECONDS=1800
AGENT_RUN_HEARTBEAT_SECONDS=30
AGENT_RUN_CANCEL_POLL_SECONDS=2
AGENT_RUN_RECOVERY_INTERVAL_SECONDS=60
RESUME_GENERATION_SESSION_TTL_HOURS=24
```

| 变量 | 作用 | 调整建议 |
| --- | --- | --- |
| `REDIS_URL` | API、Worker、模型池、BOSS 限流和审批状态使用的 Redis 地址 | 本机为 `localhost`；Compose 由服务覆盖为 `redis`。 |
| `TASK_QUEUE_ENABLED` | `true` 时 API 经 outbox/Dramatiq 派发任务 | 标准全量配置保持 `true`；`false` 仅用于单进程排障。 |
| `TASK_PAYLOAD_ENCRYPTION_KEY` | Fernet 加密任务 payload | 必填，API/Worker 一致；轮换前处理已排队任务。 |
| `AGENT_RUN_LOCK_TTL_SECONDS` | Redis 运行锁 TTL | 不要低于任务心跳与续租所需时间。 |
| `AGENT_RUN_MAX_ATTEMPTS` | 用户重试和陈旧任务恢复共用上限 | 默认 3；提高会增加外部 LLM 成本。 |
| `AGENT_RUN_STALE_SECONDS` | queued/retrying 未领取或 running 无心跳的陈旧阈值 | 太短会误恢复慢任务，太长则故障恢复慢。 |
| `AGENT_RUN_HEARTBEAT_SECONDS` | Worker 写数据库心跳频率 | 应显著小于 stale 阈值。 |
| `AGENT_RUN_CANCEL_POLL_SECONDS` | Worker 检查协作取消频率 | 越短取消越及时，但轮询更频繁。 |
| `AGENT_RUN_RECOVERY_INTERVAL_SECONDS` | API 扫描陈旧任务周期 | 不依赖用户打开任务中心。 |
| `RESUME_GENERATION_SESSION_TTL_HOURS` | 交互式简历生成会话保留时间 | 过期后不能继续追问会话。 |

Worker 启动：

```bash
cd backend
uv run dramatiq ai.runtime.agent_runs.worker --processes 1 --threads 1
```

### 3.3 LLM 网关与模型池

```dotenv

LLM_POOL_REDIS_ENABLED=true
LLM_POOL_INFLIGHT_TTL_SECONDS=600
```

这些变量控制服务端对前端 `api_config` 中模型的调用保护、超时、Token 上限、池成员失败冷却和并发计数。公开部署设 `ALLOW_PRIVATE_MODEL_BASE_URLS=false`，避免用户把后端当作内网探测代理。

### 3.4 RAG Embedding 与 Agentic Retrieval

```dotenv
OPENAI_API_KEY=
OPENAI_BASE_URL=https://api.openai.com/v1
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_DIM=1536

RAG_VECTOR_ENABLED=true
RAG_AGENTIC_MODE=shadow
RAG_AGENTIC_MAX_ROUNDS=2
RAG_AGENTIC_MAX_QUERIES=4
RAG_AGENTIC_MIN_SCORE=0.30
RAG_AGENTIC_MIN_EVIDENCES=2
RAG_AGENTIC_MIN_SOURCE_TYPES=2
RAG_WEIGHT_VECTOR=0.45
RAG_WEIGHT_TEXT=0.25
RAG_WEIGHT_SOURCE=0.20
RAG_WEIGHT_FRESHNESS=0.10
```

- `OPENAI_*` 在这里服务 Embedding，也供部分测试/兼容默认链路使用，不能替代前端主对话模型配置。
- `EMBEDDING_DIM` 必须与 embedding 模型真实输出维度一致；维度变化需要重建向量索引。
- `RAG_VECTOR_ENABLED=true` 时，`OPENAI_API_KEY`、`OPENAI_BASE_URL`、`EMBEDDING_MODEL`、`EMBEDDING_DIM` 必须完整可用。
- `RAG_AGENTIC_MODE` 取值为 `off` / `shadow` / `active`；标准配置保留该变量并显式选择模式。

### 3.5 文件解析与语音

```dotenv
FILE_PARSE_TIMEOUT_SECONDS=30

VOICE_MODEL=qwen3-omni-flash-2025-12-01
VOICE_NAME=Cherry
VOICE_INPUT_FORMAT=wav
VOICE_OUTPUT_FORMAT=wav
VOICE_TRANSCRIPT_TERM_FIXES={}
```

- 文件服务支持 PDF、DOCX、TXT；格式与大小校验在 `FileService`。
- Voice 通道的模型/API 凭据优先来自前端 `api_config.voice`；`.env` 中这些变量是服务端默认值，仍需保留。
- `VOICE_TRANSCRIPT_TERM_FIXES` 必须是 JSON 对象，例如：`{"lang graph":"LangGraph","拉姆达":"Lambda"}`。只添加确认的术语修正，避免改写候选人原话。

`MEM0_ENABLED=true` 时必须同时准备 mem0 提取 LLM、mem0 Embedding、pgvector 连接与正确维度。`MEM0_BACKGROUND_WRITE=true` 让面试后记忆写入在后台进行；`MEM0_RETENTION_DAYS` 控制记忆参与检索的保留期。Compose 会把 `MEM0_PGVECTOR_HOST` 等连接变量改为容器内 PostgreSQL。

### 3.8 BOSS 半自动化

```dotenv
BOSS_AUTOMATION_SERVICE_URL=http://host.docker.internal:8765
BOSS_AUTOMATION_SERVICE_TOKEN=
BOSS_AUTOMATION_REQUEST_TIMEOUT_SECONDS=240
JOB_CAPTURE_MAX_RESPONSE_BYTES=2097152
BOSS_BROWSER_PROFILE_DIR=
BOSS_APPLY_APPROVAL_TTL_SECONDS=300
```

- `BOSS_AUTOMATION_SERVICE_TOKEN` 至少 32 字符，主 API 与宿主机服务一致；可用 `openssl rand -hex 32` 生成。
- `BOSS_AUTOMATION_SERVICE_URL` 是 API 调用宿主机 Playwright RPC 的地址。Compose 已配置 `host.docker.internal:host-gateway`；纯本机模式按实际地址调整。
- `BOSS_BROWSER_PROFILE_DIR` 只由宿主机服务读取；为空时使用默认 `backend/data/browser_profiles/boss`，其中包含登录态。
- `BOSS_APPLY_APPROVAL_TTL_SECONDS` 是 preview 后一次性发送许可有效期；发送前必须显式 `confirmed=true`。
- 不要试图自动绕过验证码。产品流程是人工登录/验证、预览、人工确认、一次性发送。

宿主机启动命令：

```bash
cd backend
uv run playwright install chromium
uv run uvicorn app.entrypoints.boss_service:app --host 0.0.0.0 --port 8765
```

## 4. Docker Compose 与 Dockerfile 检查点

`docker-compose.yml` 不再维护一份残缺的环境变量白名单，而是通过 `env_file: .env` 将全量模板变量传入 `migrate`、`backend` 和 `worker`。Compose 只覆盖这些必须使用容器内服务名的变量：

```dotenv
DATABASE_URL=postgresql://<user>:<password>@postgres:5432/<db>
REDIS_URL=redis://redis:6379/0
AUTO_CREATE_TABLES=false
MEM0_PGVECTOR_HOST=postgres
MEM0_PGVECTOR_PORT=5432
MEM0_PGVECTOR_DBNAME=<POSTGRES_DB>
MEM0_PGVECTOR_USER=<POSTGRES_USER>
MEM0_PGVECTOR_PASSWORD=<POSTGRES_PASSWORD>
```

其它变量继续来自 `.env`，包括 `OPENAI_*`、`RAG_*`、`VOICE_*`、`LANGFUSE_*`、`BOSS_*` 和 `MEM0_*`。因此修改 `.env` 后，API/Worker 重启即可读取服务端运行变量；修改 `NEXT_PUBLIC_API_URL` 后需要重新构建前端镜像。

后端 `backend/Dockerfile` 使用同一个镜像承载 `migrate`、`backend` 和 `worker`：

- 默认命令运行 `uvicorn app.main:app`；
- Worker 通过 `/app/scripts/run-worker.sh` 启动 `ai.runtime.agent_runs.worker`；
- 镜像 healthcheck 使用 `python -m app.entrypoints.deployment readiness` 校验 PostgreSQL schema 与 Redis；
- `migrate` 服务禁用 healthcheck，避免一次性迁移容器被长期健康检查误判。

## 5. 启动与配置验证

```bash
# 本机依赖
docker compose --env-file .env up -d postgres redis

# 已管理的本地数据库迁移
cd backend && uv run alembic upgrade head

# API/Worker
cd backend && uv run python -m uvicorn app.main:app --reload
cd backend && uv run dramatiq ai.runtime.agent_runs.worker --processes 1 --threads 1

# 完整基础设施 readiness
cd backend && uv run python -m app.entrypoints.deployment readiness
```

`/health` 只检查 API HTTP 进程；不代表 PostgreSQL、Redis、LLM、BOSS 都可用。模型连接用前端设置页测试或 `POST /api/config/validate`；队列状态从 `/api/agent-runs/{run_id}/events` / SSE 判断。

## 6. 安全清单

- `.env`、浏览器模型配置、BOSS profile、数据库备份均可能包含秘密或个人数据；不要提交或共享。
- 生产中替换演示身份头 `X-User-ID` 为真实认证/授权；它不是安全的用户认证机制。
- 公开部署设置 `ALLOW_PRIVATE_MODEL_BASE_URLS=false`，并由外部 TLS 终止代理提供 HTTPS。
- 不在日志、错误响应、截图、AgentRun payload 或文档中打印模型/BOSS/数据库密钥。
- BOSS 发送属于真实外部副作用，始终保留人工 preview 与确认步骤。
