## 2. 快速启动

### 2.1 启动 PostgreSQL（Docker）

```bash
# 在项目根目录执行
docker-compose up -d db
```

这会在 `localhost:5432` 启动 PostgreSQL，并自动创建 `agent_interview` 数据库和 `pgvector` 向量扩展。

### 2.2 配置环境变量

```bash
# 复制示例配置
cp env_example .env

# 编辑 .env，填入你的 API Key
# 必填项：LLM 相关配置（至少配一个模型通道）
```

### 2.3 安装依赖 & 启动后端

```bash
cd backend

# 安装 Python 依赖
uv sync

# 运行数据库迁移
uv run alembic upgrade head

# 启动后端服务（开发模式）
uv run uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

### 2.4 启动前端

```bash
cd frontend

# 安装前端依赖
npm install

# 启动开发服务器
npm run dev
```

访问：
- 后端 API 文档：http://localhost:8000/docs
- 前端界面：http://localhost:3000

## 3. 环境变量详解

所有配置通过项目根目录的 `.env` 文件进行管理。

### 3.1 数据库配置

```bash
# PostgreSQL 数据库地址（完整 DSN）
DATABASE_URL=postgresql://agent_interview:cheng123@localhost:5432/agent_interview

# 或者分别指定：
POSTGRES_USER=agent_interview       # 数据库用户名
POSTGRES_PASSWORD=cheng123          # 数据库密码
POSTGRES_DB=agent_interview         # 数据库名称
POSTGRES_HOST=localhost             # 数据库主机
POSTGRES_PORT=5432                  # 数据库端口
```

> **优先级**：如果设置了 `DATABASE_URL`，则优先使用；否则从 `POSTGRES_*` 变量拼接。

### 3.2 LLM 模型配置

```bash
# --- 服务端默认配置（前端未配置时生效）---
OPENAI_API_KEY=sk-xxx                            # 默认 API Key
OPENAI_BASE_URL=https://api.openai.com/v1         # 默认 API 地址
SMART_MODEL=gpt-4o                                # Smart 通道模型
FAST_MODEL=gpt-4o-mini                             # Fast 通道模型
```

> **推荐做法**：不在 `.env` 中配置，而是在前端设置页面的「模型配置」中分别配置各通道的 API Key。`.env` 中的配置仅作为服务端兜底。

### 3.3 前端配置

```bash
NEXT_PUBLIC_API_URL=http://localhost:8000   # 后端 API 地址
```

### 3.4 mem0 长期记忆配置

mem0 是项目的长期记忆系统，记住用户偏好和面试历史。

```bash
# 功能开关
MEM0_ENABLED=true                    # 设为 false 可禁用

# 向量存储（复用 PostgreSQL）
MEM0_VECTOR_PROVIDER=pgvector
MEM0_PGVECTOR_COLLECTION=mem0_memories
MEM0_PGVECTOR_HOST=localhost
MEM0_PGVECTOR_PORT=5432
MEM0_PGVECTOR_DBNAME=agent_interview
MEM0_PGVECTOR_USER=agent_interview
MEM0_PGVECTOR_PASSWORD=cheng123

# mem0 使用的 LLM（用于记忆提取）
MEM0_LLM_PROVIDER=openai
MEM0_LLM_MODEL=deepseek-chat
MEM0_LLM_API_KEY=sk-d93fef172a3c4a709b2222285673be33
MEM0_LLM_BASE_URL=https://api.deepseek.com/v1

# mem0 Embedding（用于语义检索）
MEM0_EMBEDDER_PROVIDER=openai
MEM0_EMBEDDER_MODEL=tongyi-embedding-vision-plus-2026-03-06
MEM0_EMBEDDER_API_KEY=sk-95fe6cd909f6473abcf6398b811c3e39
MEM0_EMBEDDING_DIMS=1536

# 运行时参数
MEM0_SEARCH_LIMIT=5                  # 每次搜索返回的记忆条数
MEM0_CONTEXT_CHAR_LIMIT=1200         # 上下文最大字符数
MEM0_BACKGROUND_WRITE=true           # 是否后台写入记忆
```

> **关键配置**：`MEM0_LLM_MODEL` 和 `MEM0_LLM_API_KEY` 可以指向便宜的模型（如 deepseek-chat），用于记忆提取，不消耗主力模型的 tokens。

## 4. LLM 多通道配置

项目支持多通道独立配置，不同任务使用不同的模型，以平衡性能和成本。

### 4.1 通道说明

| 通道 | 用途 | 推荐模型 | 要求 |
|------|------|---------|------|
| **smart** | 复杂任务（面试官回复、分析点评、总结） | qwen-max / gpt-4o / deepseek-chat | 高智商 |
| **fast** | 快速响应（意图识别、简单分类） | qwen-flash / gpt-4o-mini | 低延迟 |
| **general** | 通用任务（简历分析、开场白） | 同 smart | 中等 |
| **match_analyst** | JD 匹配分析 | 同 smart 或专用模型 | 分析能力强 |
| **content_writer** | 内容优化（简历润色） | 同 smart | 写作能力强 |
| **hr_reviewer** | HR 审核（面试模拟终审） | 同 smart | 评判客观 |
| **reflector** | 质量审核（自我反思） | 同 smart | 批判性思维 |

### 4.2 Fallback 策略

每个通道查询时的回退链：

```
请求通道 → general → smart
```

例如：请求 `match_analyst` 通道时，如果未配置，则依次尝试 `general` → `smart`。如果 `smart` 也未配置，则抛出异常。

### 4.3 前端配置方式（推荐）

在前端「设置 → 模型配置」中为每个通道分别填入：

- **API Key**：模型的 API 密钥
- **Base URL**：API 兼容接口地址（OpenAI 兼容格式）
- **Model**：模型名称

配置会随请求发送到后端，后端动态创建 LLM 实例。

## 6. Docker 服务编排

`docker-compose.yml` 定义了后端依赖服务：

```yaml
services:
  db:
    image: pgvector/pgvector:pg16        # PostgreSQL 16 + pgvector
    environment:
      POSTGRES_USER: agent_interview
      POSTGRES_PASSWORD: cheng123
      POSTGRES_DB: agent_interview
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
```

启动/停止：

```bash
# 启动
docker-compose up -d db

# 停止
docker-compose down

# 查看日志
docker-compose logs -f db
```

> **注意**：服务端口与本地已安装的 PostgreSQL 可能冲突。修改 `ports` 映射（如 `5433:5432`）然后更新 `.env` 中的端口即可。

## 7. 常见问题

### Q: 启动后端时报错 `connection refused`？

A: 确保 PostgreSQL 已启动。检查 `docker-compose ps` 或本地 PostgreSQL 服务状态。

### Q: LLM 调用返回 401 / Unauthorized？

A: 检查 API Key 是否正确、是否过期。可以在前端「设置 → 模型配置」中点击「验证」按钮测试连通性。

### Q: mem0 记忆功能不工作？

A: 检查 `MEM0_ENABLED=true`，并确认 mem0 的 LLM 和 Embedding 配置正确。可以临时设置 `MEM0_ENABLED=false` 来绕过问题。

### Q: 数据库表不存在？

A: 执行数据库迁移：`cd backend && uv run alembic upgrade head`

### Q: 如何只启用部分通道？

A: 最少配置 `smart` 通道即可，其他通道会自动 fallback 到 `smart`。不需要为每个通道都配置单独的模型。
