> 更新日期：2026-07-16

## 项目演进背景与原始审视结论

本项目最初从功能型 AI 求职助手逐步演进为包含多 Agent、模型池、后台任务、长期记忆、RAG 和浏览器工具的模块化单体。

架构审视时的核心结论：

| 维度 | 当时评价 | 主要问题 |
|---|---|---|
| 目录与物理模块 | 较好 | 已有 API、Service、Repository、Agent Runtime 等目录 |
| 高内聚 | 中等 | 多个 800–1450 行巨型模块，单文件承担编排、模型、存储和协议 |
| 低耦合 | 偏弱 | API 直接访问 Repository，Repository 反向依赖 Service，Runtime 与业务双向依赖 |
| 领域边界 | 中等 | 按产品功能分目录，但缺少明确 Application Use Case 和事务边界 |
| 可靠性 | 中上 | 已有幂等、任务恢复和锁，但 checkpoint、取消、事件重放仍不完整 |
| Agent Runtime | 偏弱 | Registry、Middleware、Tool Guard 已存在，但生产业务接入不足 |
| 生命周期 | 偏弱 | AgentRun、SSE 请求和 LangGraph 分别使用不同生命周期 |
| 事件机制 | 较弱 | 多条 SSE 协议各自实现，没有统一信封、sequence 和重放 |
| 模型调度 | 较好 | 模型池能力较完整，但选择预占不原子、普通文本 fallback 覆盖不全 |
| 测试 | 中上 | 单元和回归较多，但真实 PostgreSQL/Redis/Checkpoint 集成测试不足 |

原始审视中确认的关键缺陷包括：

1. 同步 `PostgresSaver` 被当成异步对象使用，实际长期回退到 `MemorySaver`；
2. 面试 Graph 实例由全局强引用列表永久持有；
3. 陈旧 AgentRun 只有用户打开任务中心时才恢复；
4. 运行中的 AgentRun 无法取消；
5. SSE 没有事件 ID、sequence、版本和断线重放；
6. Agent Runtime 的 Factory、GraphRegistry、PromptRegistry 等多数只在测试中使用；
7. Redis 模型选择与 in-flight 增加存在竞争窗口；
8. 普通文本模型调用大量只取首候选，没有 fallback；
9. Repository 存在反向依赖 Service；
10. API 层承担较多跨 Repository 和跨领域编排。

基于以上结论，项目确定继续采用：

> 模块化单体 + Application Use Case + 统一 Agent Runtime + 持久化 RunEvent

不采用微服务拆分，也不在个人本地项目中增加登录认证、JWT/OAuth 或公网部署体系。

- API 不同时调用两个以上 Repository；
- 跨领域流程通过 Application Use Case 编排；
- Repository 只负责持久化；
- 业务异常由应用层定义，再映射为 HTTP 状态码。

# 4. P1：交互式 Agent 生命周期统一

## 4.1 当前差异

当前有三套生命周期：

```text
AgentRun：queued/running/retrying/succeeded/failed/cancelled
SSE 请求：request → delta → done/error
LangGraph：compile → invoke → checkpoint
```

AgentRun 主要覆盖后台任务，文字和语音面试仍主要依赖 HTTP/SSE 请求生命周期。

## 4.2 待办

- [x] 为文字面试的每次生成建立持久化 Run；
- [x] 为语音面试生成建立持久化 Run；
- [ ] 定义 parent/child run，例如面试 Run 下包含模型调用和报告子 Run；
- [x] 将浏览器断开与 Agent 取消区分开；
- [x] 页面重新连接时根据 RunEvent 恢复中间状态；
- [ ] 统一 `pause/resume/cancel_requested/cancelled` 语义；
- [ ] 为需要人工确认的外部工具增加 `awaiting_approval`；
- [ ] 恢复后继续使用同一 checkpoint namespace；
- [x] 明确哪些 Run 允许整次重试，哪些只允许从 checkpoint 恢复。

## 4.3 建议状态机

```text
created
  ↓
queued
  ↓
running ───────────────→ succeeded
  │                         │
  ├→ awaiting_approval ─────┘
  ├→ pause_requested → paused → running
  ├→ cancel_requested → cancelled
  └→ failed → retrying → queued
```

## 4.4 完成标准

- 所有重要 Agent 调用都有 `run_id`；
- 断开 SSE 不会隐式取消 Agent；
- 运行中可以协作取消；
- 重启后可以从数据库和 checkpoint 判断真实状态；
- 页面可以通过事件序列恢复进度；
- 一个 Run 的所有模型、工具和产物事件可关联查询。
