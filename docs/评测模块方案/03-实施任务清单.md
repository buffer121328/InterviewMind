# 评测模块 — 实施任务清单

## 总览

分 3 个阶段实施，共 28 个任务。每个任务包含：目标、具体工作、验收标准、预估耗时。

| 阶段 | 任务数 | 预估总耗时 | 核心产出 |
|------|--------|-----------|---------|
| 阶段 1：基础设施 + 工作流正确性 | 10 | 1-2 周 | pytest 框架 + 工作流测试 + 回归测试 |
| 阶段 2：LLM 输出质量评测 | 10 | 2-3 周 | DeepEval 集成 + 质量指标 + Golden Dataset |
| 阶段 3：高级评测 + 持续优化 | 8 | 3-4 周 | 评分校准 + A/B 测试 + Dashboard |

## 阶段 1：基础设施 + 工作流正确性（1-2 周）

> 目标：建立评测框架，跑通第一批测试，0 LLM token 消耗

### T1.1 依赖引入与环境配置

**目标**：项目能运行 `pytest tests/eval/`

**具体工作**：
1. 在 `backend/pyproject.toml` 添加 `eval` 依赖组：
   ```toml
   [project.optional-dependencies]
   eval = [
       "deepeval>=2.0.0",
       "pytest-asyncio>=0.23.0",
       "pytest>=8.0.0",
   ]
   ```
2. 创建 `backend/pytest.ini`：
   ```ini
   [pytest]
   testpaths = tests
   markers =
       fast: 快速测试（不消耗 LLM tokens）
       llm: 需要 LLM API 的测试
       regression: 回归测试套件
       eval: 质量评测
   asyncio_mode = auto
   ```
3. 运行 `uv sync --extra eval` 验证安装

**验收标准**：
- [ ] `uv sync --extra eval` 无报错
- [ ] `python -m pytest --co tests/eval/` 能发现测试文件（即使还没有测试）

**预估耗时**：0.5h

   ```python

   from deepeval.test_case import LLMTestCase

   def test_deepeval_basic():
       metric = AnswerRelevancyMetric(threshold=0.5)
       test_case = LLMTestCase(
           input="什么是 Python？",
   ```
