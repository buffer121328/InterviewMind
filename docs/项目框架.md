## 核心模块说明

### 1. LangGraph 面试状态机 (`interview_graph.py`)

面试流程的核心控制器，管理对话状态和流程转换。

```python
# 状态定义
class InterviewState(TypedDict):
    messages: List[BaseMessage]      # 对话历史
    question_count: int              # 当前题号
    max_questions: int               # 总题数
    current_question: str            # 当前问题
    mode: Literal["coach", "mock"]   # 面试模式
    is_finished: bool                # 是否结束

# 流程节点
# 1. opening_node    - 开场白生成
# 2. question_node   - 提问生成
# 3. analyze_node    - 回答分析
# 4. finish_node     - 结束总结

# 路由逻辑
# should_continue() - 判断继续提问/结束面试
```

### 2. 会话服务 (`app/repositories/session/`)

管理面试会话的完整生命周期。

```python
class SessionService:
    async def create_session(...)      # 创建新会话
    async def get_session(...)         # 获取会话详情
    async def list_sessions(...)       # 列出所有会话
    async def update_session(...)      # 更新会话元数据
    async def delete_session(...)      # 删除会话
    async def add_message(...)         # 添加消息
    async def save_ability_profile(...)# 保存能力档案
```

### 3. LLM 客户端 (`llms.py`)

统一管理 LLM 调用，支持动态配置。

```python
class LLMClientFactory:
    def get_smart_model(config)        # 获取智能模型 (GPT-4)
    def get_fast_model(config)         # 获取快速模型 (GPT-3.5)
    
# 支持:
# - OpenAI 官方 API
# - Azure OpenAI
# - 兼容 API (如 deepseek, 通义千问等)
```

### 4. 前端状态管理 (`useInterviewStore.ts`)

Zustand store 管理全局面试状态。

```typescript
interface InterviewStore {
  // 状态
  sessions: Session[]
  currentSession: Session | null
  messages: Message[]
  isStreaming: boolean
  
  // 操作
  createSession: () => Promise<void>
  loadSession: (id: string) => Promise<void>
  sendMessage: (content: string) => Promise<void>
  startInterview: (config: InterviewConfig) => Promise<void>
}
```

### 会话 (Session)

```json
{
  "session_id": "uuid",
  "title": "Python 后端面试",
  "mode": "coach",
  "status": "active",
  "question_count": 3,
  "max_questions": 5,
  "created_at": "2025-12-05T10:00:00",
  "updated_at": "2025-12-05T10:30:00"

```

### 消息 (Message)

```json
{
  "id": "msg-uuid",
  "role": "assistant",
  "content": "请介绍一下你对 Python 异步编程的理解。",
  "timestamp": "2025-12-05T10:15:00"

```

## 通信协议

### REST API

标准 RESTful 接口，使用 JSON 格式。

### SSE 流式响应

面试对话使用 Server-Sent Events 实现流式输出：

```
GET /api/chat/stream

event: token
data: {"content": "请"}

event: token
data: {"content": "介绍"}

event: done
data: {"full_response": "请介绍一下..."}
```
