## 15. 这个项目可以怎么写进简历

项目名称：

```text
InterviewMind｜AI 智能面试助手
```

项目描述：

> InterviewMind 是一个面向求职者的 AI 面试训练与简历优化平台，支持模拟面试、语音面试、简历分析、JD 匹配和投递追踪等功能，帮助用户完成从面试训练到简历优化的求职准备闭环。

---

## 16. 简历核心功能写法

可以写：

- 基于 FastAPI 构建后端服务，支持文字面试、语音面试、简历分析、JD 匹配、题库管理和投递追踪等功能。
- 使用 LangGraph 设计面试流程状态机，支持 SSE 流式输出与多轮面试上下文继承。
- 基于 PostgreSQL + pgvector + mem0 构建长期记忆与 RAG 检索能力，实现用户偏好、候选人事实、短板和练习目标的语义检索与上下文注入。
- 实现 Agentic RAG，将题库、历史面试记录、候选人素材和短板报告作为出题依据，提升面试问题的岗位匹配度和个性化程度。
- 支持简历竞争力分析、JD 匹配分析、项目经历重写、简历生成和投递进度管理，形成较完整的求职辅助流程。

## 18. 面试时怎么讲后端架构

你可以这样说：

> 后端采用 FastAPI 构建 API 服务，使用 Pydantic 做数据校验，SQLAlchemy Async ORM 操作 PostgreSQL。整体按 API、Service、Repository、Model、Schema 分层，避免接口层直接耦合数据库逻辑。会话模块使用 Facade 模式封装多个子服务，简化业务调用。

---

## 19. 面试时怎么讲 AI 工作流

可以这样说：

> 面试功能基于 LangGraph 实现状态机，将面试流程拆成 planner、responder、summary 三类节点。planner 根据简历、JD、轮次策略和 RAG 检索结果生成面试计划，responder 负责模拟面试官进行追问和推进，summary 在面试结束后生成反馈并触发画像分析。

---

## 20. 面试时怎么讲 RAG

可以这样说：

> 项目里的 RAG 主要用于面试题生成增强。系统会将题库、候选人素材、历史面试记录和短板报告向量化索引到 PostgreSQL + pgvector 中，在新一轮面试开始时按 JD、轮次类型和候选人薄弱项检索相关证据，并注入到 planner prompt 中，从而提升问题生成的岗位匹配度和个性化程度。

---

## 21. 面试时怎么讲长期记忆

可以这样说：

> 项目使用 mem0 构建长期记忆系统，保存用户偏好、候选人事实、短板和练习目标。面试开始和用户回答时，系统会根据当前 JD 和回答内容检索相关记忆，并将其格式化后注入 prompt，使 AI 面试官能够结合用户历史表现进行更个性化的提问。

---

## 22. 当前项目的不足

真实面试时，也可以主动说不足：

- 用户认证还比较简单，目前主要基于 `X-User-ID`
- 后台任务还是轻量进程内任务，可靠性不如任务队列
- 自动化测试还可以继续补充
- RAG 评估体系还不完善
- 多用户权限模型可以更细

能说出不足，反而说明你真的理解项目。

## 24. 初学者学习路线

建议按 6 个阶段学习。

### 阶段 1：Python Web 基础

学习：

- FastAPI
- Pydantic
- HTTP 请求
- REST API
- 异常处理

练习：

- 写 3 个 CRUD 接口
- 用 Swagger 和 curl 测试

---

### 阶段 2：数据库

学习：

- PostgreSQL
- SQLAlchemy
- async/await
- 表关系
- 索引

练习：

- 新增一张表
- 写 Repository
- 实现增删改查

---

### 阶段 3：后端分层

学习：

- API 层
- Service 层
- Repository 层
- Schema 层
- Model 层

练习：

- 把一个接口从“全部写在 API 中”重构为分层结构

---

### 阶段 4：LLM 应用

学习：

- Prompt
- LangChain message
- 结构化输出
- LLM fallback
- 流式输出

练习：

- 写一个简历评分接口
- 返回结构化 JSON

---

### 阶段 5：Agent 和 RAG

学习：

- LangGraph
- StateGraph
- RAG
- embedding
- pgvector
- tool calling

练习：

- 做一个小型知识库问答
- 再把检索结果用于生成面试题

---

### 阶段 6：工程化

学习：

- Docker
- Nginx
- 环境变量
- 日志
- 测试
- 部署

练习：

- 用 Docker Compose 启动完整项目
